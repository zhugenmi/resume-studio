# 隐私脱敏规则

简历含个人隐私信息。导出、分享、作为样例展示前必须脱敏。本 skill 不替用户决定分享边界，但生成可公开样例或对外分享诊断报告时必须遵守本规则。

## 必脱敏字段

| 字段 | 脱敏方式 |
|------|---------|
| 真实姓名 | 替换为 `{{姓名}}` 或化名（张三/李四） |
| 手机号 | 保留前 3 后 4，中间 4 位 `*`：`138****5678` |
| 邮箱 | 用户名首字符 + `***` + 域名：`z***@example.com` |
| 身份证/护照 | 完全删除，不脱敏保留 |
| 现任职公司名（敏感场景） | 替换为 `A 公司`/`B 公司` 或行业泛称 |
| 家庭住址 | 仅保留城市级：`北京` 而非完整地址 |
| 紧急联系人 | 完全删除 |

## 可保留字段（非敏感）

- 工作年限、目标岗位、城市意向
- 技术栈、项目经历（除非项目本身涉密）
- 学历、毕业院校（除非用户要求隐藏）
- GitHub/LinkedIn 链接（公开主页，除非用户要求隐藏）

## 何时触发脱敏

1. **用户要求「分享/公开/发到社区」诊断报告或简历样例** → 必脱敏。
2. **本 skill 生成 examples/ 回归样例** → 必脱敏。
3. **用户只是本地查看** → 不脱敏，原样输出。
4. **导出 PDF/HTML 投递** → 不脱敏（投递需要真实信息），但提示用户核对不要误发错对象。

## 脱敏后自检

脱敏后扫一遍产物，确认：
- 无完整手机号、邮箱、身份证号残留。
- 无真实姓名与现任职公司同时出现（防交叉定位）。
- 项目描述里的真实用户/客户名替换为「某 X 公司」「某 Y 平台」。

## 用户提示

生成可分享版本时附一句：
> 已脱敏处理（姓名/联系方式/现任公司已掩码）。分享前请再核对一遍，确保没有遗漏的隐私信息。
