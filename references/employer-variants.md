# 雇主版本维度（soe / internet / foreign）

简历差异不止「长度」与「风格」，还有「目标雇主类型」。同一份经历，投国企/银行、私企/互联网、外企，评分标准、必含信息、风险红线、语言都不一样。本文件是三版差异的**单一事实源**，其余 references（scoring/audit/red-flags/export/workflow）通过指针引用此处。

版本代号固定：`soe`（国企/银行）、`internet`（私企/互联网）、`foreign`（外企）。

## 两维正交

- **雇主版本维度**（本文件）：soe / internet / foreign —— 决定评分标准、必含信息、风险红线、语言。
- **长度维度**（`resume-variants.md`）：详尽 / 标准 / 一行 —— 决定每段经历篇幅。

同一版内仍按 `resume-variants.md` 取长度。风格维度（HTML 模板）由版本默认映射决定，用户可覆盖。

## 三版总览

| 版本 | 代号 | 语言 | 默认模板 | 目标雇主 |
|------|------|------|---------|---------|
| 国企/银行版 | `soe` | 中文 | soe.html | 央企/国企/银行/事业单位/政策性机构 |
| 私企/互联网版 | `internet` | 中文 | internet.html | 民营科技公司/互联网大厂/创业公司 |
| 外企版 | `foreign` | 英文（可切中文） | foreign.html | 跨国公司/外资企业/海外岗 |

模块取舍见 `resume-modules.md`。

## soe — 国企/银行版「稳、全、合规」

### 必含信息
- 政治面貌（党员/预备党员优先，按岗位要求标注）
- 籍贯 / 生源地
- 正装证件照（非生活照）
- 出生年月
- 证书（CPA/CFA/银行从业/法律职业资格等岗位相关证）

### 侧重
- 稳定性叙事：避免频繁跳槽，空窗期需合理说明。
- 校园经历 / 学生干部 / 奖学金（校招/应届必查）。
- 家庭背景（部分岗位要求）。
- 格式严谨、信息完整、合规。

### 风险红线
- 政治面貌与岗位硬要求冲突（如非党员投党员优先岗）→ P0，建议放弃或转岗。
- 跳槽频繁（平均在职 <2 年）且无解释 → 稳定性红旗，扣影响力分。
- 创业/外包经历未重新叙述 → 标注需重述为「交付能力」。

## internet — 私企/互联网版「快、准、量化」

### 必含信息
- GitHub / 作品集链接（技术岗必备）
- 技术栈
- 项目上下文先行（每段项目一句话描述：系统定位 + 服务对象 + 业务问题）
- 强量化

### 侧重
- STAR + 强动词。
- 业务影响数据（非仅工程数据）。
- JD 关键词命中（ATS 初筛重）。
- 近 1-2 段深挖。

### 风险红线
- 玩具项目冒充生产项目 → P0 删除或补真实用户/规模。
- 全篇无业务量化 → P1 按 `star-rewrite.md` 补。
- 「精通」无项目证据 → P1 降级为熟悉/了解。

## foreign — 外企版「achievement-led, concise, leadership」

### 必含信息
- quantified achievements（动词开头）
- leadership / collaboration 证据

### 侧重
- **全英文**（默认，无中文字符残留；用户明确要中文才切）。
- 英文 action verbs（Led / Spearheaded / Architected / Built / Shipped）。
- 删照片 / 年龄 / 婚否 / 籍贯（合规：防歧视）。
- 一页原则：社招强制 1 页，应届 ≤1 页。
- cover letter 友好。

### 风险红线
- 含照片 / 年龄 / 性别 / 婚否 → P0 合规硬伤，必须删。
- 中式自评（hardworking / earnest 类空洞词）→ P1 删除或改 achievement。
- bullet 无 action verb 开头 → P1 重写每条 bullet。
- 超 1 页（社招）→ P1 压缩。

## 评分 overlay — 维度权重偏移

100 分 / 5 维度骨架不变（见 `scoring-rubric.md`），但每版权重偏移。合计仍 100。

| 维度 | 基础 | soe | internet | foreign |
|------|------|-----|----------|---------|
| 内容质量 | 30 | 25 | 32 | 30 |
| 结构排版 | 25 | 30 | 23 | 25 |
| 语言语法 | 20 | 20 | 18 | 22 |
| ATS 优化 | 15 | 10 | 17 | 15 |
| 影响力 | 10 | 15 | 10 | 8 |
| **合计** | 100 | 100 | 100 | 100 |

权重逻辑：
- **soe**：重结构严谨与稳定印象（结构↑影响力↑），ATS↓因国企多人工筛选。
- **internet**：重内容量化与 ATS 关键词（内容↑ATS↑），结构略↓因互联网容忍灵活排版。
- **foreign**：重英文地道与精炼（语言↑），影响力略↓因一页约束下信息密度优先。

**分数仅同版纵向对比**（基线 → 优化后），禁止跨版比——标准不同。

## 版本专属审计项

叠加在通用 8 大类（见 `audit-checklist.md`）之上。

### soe 专属
- [ ] 政治面貌已写明（党员/预备党员标注）
- [ ] 籍贯/生源地已填
- [ ] 正装证件照（非生活照）
- [ ] 出生年月已填
- [ ] 证书齐全（岗位相关证）
- [ ] 跳槽频率可控（平均在职≥2年，或附合理解释）
- [ ] 校园经历/学生干部/奖学金（校招/应届必查）
- [ ] 无创业/外包等「不稳定」标签未解释

### internet 专属
- [ ] GitHub/作品集链接（技术岗必备）
- [ ] 每段项目有上下文描述（系统定位+服务对象）
- [ ] 关键经历含量化（业务影响数据，非仅工程数据）
- [ ] JD 硬技能词原样出现
- [ ] 无玩具项目/课程项目冒充生产项目
- [ ] 技术栈与项目证据一致（「精通」有支撑）

### foreign 专属
- [ ] 全英文（无中文字符残留）
- [ ] 无照片/年龄/性别/婚否/籍贯（合规防歧视）
- [ ] 每条 bullet 以英文 action verb 开头（Led/Spearheaded/Architected/Built）
- [ ] 社招强制 1 页（应届≤1 页）
- [ ] quantified achievements（每段关键经历有数字）
- [ ] 无中式自评（"hardworking/earnest"类空洞词）
- [ ] leadership/collaboration 证据（非单打独斗）

## 文件命名规则

所有产物文件名带版本后缀，落 `resume/` 目录：

| 文件 | 命名 |
|------|------|
| 简历工作版 | `resume-<variant>.md` |
| 简历 HTML | `resume-<variant>.html` |
| 诊断报告 | `resume-diagnosis-<variant>.md` |
| JD 分析 | `jd-analysis-<variant>.md` |
| 面试问答 | `interview-qa-<variant>.md` |
| 一页版 | `resume-<variant>-one-page.md` |

批量模式（一次三版）：顺序跑三遍，产出 `resume-soe.*`、`resume-internet.*`、`resume-foreign.*` 三套，每版各一份诊断 + 各一份面试问答，不合并。
