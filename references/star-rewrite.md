# STAR 改写公式与强动词表

## STAR 法则

把「负责 XX」改成 STAR 完整结构：

- **S 情境（Situation）**：在什么背景/问题下
- **T 任务（Task）**：要解决什么
- **A 行动（Action）**：具体做了什么（用强动词）
- **R 结果（Result）**：带数字的产出

### 改写示例

**原文（职责型）**：
> 负责新媒体运营

**改写（STAR + 量化）**：
> 独立运营公司公众号（S/T），通过内容选题优化与每周 A/B 测试（A），3 个月内将粉丝数提升 50%、打开率从 8% 提至 15%（R）。

**原文**：
> 参与支付系统开发

**改写**：
> 主导支付核心模块重构（A），将接口延迟从 800ms 降到 200ms（R），支撑日均 50 万笔交易（S 规模）。

## 项目描述模板（上下文先行）

每段项目经历必须先用一句话说明项目背景，再写 bullet：

```text
项目名称（你的角色）
一句话项目描述：[系统定位] + [核心用户/客户] + [解决的业务问题]
- bullet 1: 动作 + 产物 + 结果
- bullet 2: ...
```

缺少项目描述时用占位符：

```text
[项目描述待补：例如面向XX用户的XX系统，用于解决XX问题]
```

## 强动词替换表

| 避免 | 改用 |
|------|------|
| 负责 | 主导 / 牵头 / 统筹 / 担纲 |
| 参与 | 协同 / 联合 / 配合 |
| 做了 | 重构 / 落地 / 上线 / 交付 |
| 帮助 | 赋能 / 支撑 / 助力 |
| 优化 | 精简 / 加速 / 降本 |
| 改进 | 迭代 / 升级 / 改造 |
| 完成 | 达成 / 实现 / 交付 |
| 处理 | 解决 / 攻克 / 排查 |

## 量化路径（无准确数字时）

无法给精确数字时，用以下维度证明价值：

- **范围**：覆盖 X 个团队 / Y 个系统
- **频率**：日均 / 周均 / 每月
- **规模**：服务 X 万用户 / Y 亿数据
- **效率**：耗时从 X 降到 Y / 提速 N 倍
- **风险**：故障率降低 X% / 0 事故
- **流程**：从手动到自动化 / 从 X 天到 Y 天

## 改写纪律

1. 忠于原文，不凭空扩写经历
2. 优先改写最关键模块：摘要、最近两段工作、最相关项目
3. 缺数据标占位符 `[量化指标待补：...]`，不编造
4. 每条关键 bullet 体现"动作 + 产物 + 结果"
5. 每段项目开头必须有项目描述
