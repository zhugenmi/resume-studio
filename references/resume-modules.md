# 简历模块与取舍规则

简历由 9 个常见模块组成。每个模板（`templates/soe.html` / `internet.html` / `foreign.html`）均含全 9 模块脚手架，填充时按**目标雇主版本** + **用户自身情况**取舍——无该经历则删整段（含标题），不留空标题。本文件是模块取舍的**单一事实源**，被 `export-templates.md` / `employer-variants.md` / `workflow.md` 引用。

## 9 模块定义

| 代号 | 模块 | 说明 |
|------|------|------|
| M1 | 基础信息/联系信息 | 姓名/联系方式 + 版本专属字段 |
| M2 | 职业摘要/自我评价 | 价值主张或研究兴趣 |
| M3 | 教育背景 | 学校/专业/学位/时间，可选 GPA/课程/奖学金 |
| M4 | 工作/实习经历 | STAR + 强动词 |
| M5 | 项目经历 | 含研究经历，项目上下文先行 |
| M6 | 竞赛/奖项/荣誉 | 校招重点 |
| M7 | 实践/校园经历 | 学生干部/社团/志愿，体现 leadership |
| M8 | 出版物/论文 | 科研岗可选 |
| M9 | 专业技能 | 含证书 + 语言能力，分小节 |

> 模块合并约定：研究经历并入 M5 项目经历（按项目结构写：背景/方法/产出）；证书与语言能力并入 M9 专业技能（分「核心技能/证书/语言」小节）。学术模板已删除，科研岗投外企/互联网时用 foreign/internet 模板 + M8 出版物。

## 模块取舍矩阵

| 模块 | soe（国企/银行） | internet（私企/互联网） | foreign（外企·英文） |
|------|------|------|------|
| M1 基础信息 | 必含：政治面貌/籍贯/生源地/出生年月/正装照 | 必含：GitHub/作品集 | 必含，**禁**照片/年龄/性别/婚否/籍贯 |
| M2 摘要 | 可选，稳定叙事 | 必含，价值主张+量化 | 必含，value proposition |
| M3 教育 | 必含，课程/GPA/奖学金 | 必含，应届重点 | 必含 |
| M4 工作/实习 | 必含（如有），稳定合规 | 必含，STAR+业务量化 | 必含，action verb+quantified |
| M5 项目经历 | 可选（含研究） | 必含，上下文先行 | 可选（含研究） |
| M6 竞赛/奖项 | 必含（校招重点） | 可选（应届/算法岗） | 可选 |
| M7 实践/校园 | 必含（校招，学生干部） | 可选（应届可写） | 可选，体现 leadership |
| M8 出版物 | 一般无 | 可选（科研岗） | 可选 |
| M9 专业技能 | 必含（含证书 CET-4/6、CPA） | 必含（技术栈+云认证） | 必含（技能+证书+语言） |

## 按用户自身情况取舍（正交于版本）

版本决定「该模块必含还是可选」；用户情况决定「可选模块留不留」「必含模块怎么写」。

| 用户情况 | 取舍 |
|---------|------|
| 应届生无工作/实习 | 删 M4，强化 M3/M5/M6/M7 |
| 应届生有实习 | M4 写实习 |
| 社招 | 删 M6/M7（除非强相关），强化 M4 |
| 转行 | 强化 M9 可迁移技能 + M5 相关项目，弱化无关 M4 |
| 无项目经历 | 删 M5，M4 内项目化描述 |
| 算法/科研岗 | M5 含研究 + 加 M8 |

## 填充纪律

1. **删整段不留空标题**：无该经历则删除整个模块（含 `<h2>` 标题与内容），简历里不出现空段。
2. **模块顺序固定**：按 M1→M2→M3→M4→M5→M6→M7→M8→M9 排列，不随意调换（教育背景在工作经历前为校招惯例；社招可 M4 在 M3 前，但模板默认顺序不调，由填充时移动）。
3. **模板内注释保留到投递前删除**：每个模块前的 `<!-- Mx ... -->` 注释是填充指引，填充完成后投递版必须删除所有 HTML 注释。
4. **占位符全替换**：`{{...}}` 占位符必须全部替换为真实内容，无数据用 `[量化指标待补：...]` 标注，不编造。
5. **版本专属字段不可漏**：soe 的政治面貌/籍贯/证书、internet 的 GitHub/项目上下文、foreign 的合规字段（删隐私）+ action verb 是版本硬约束，缺失即扣分。
