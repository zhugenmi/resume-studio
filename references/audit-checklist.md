# 审计检查清单（40+ 项，8 大类）

逐类给 ✅（通过）/ ⚠️（待改进）/ ❌（致命）。致命项必须改，待改进项建议改。

## A. 联系信息（5 项）

- [ ] 邮箱格式规范（非 QQ 昵称类随意邮箱）
- [ ] 电话号码正确且可联系
- [ ] LinkedIn / GitHub / 作品集链接（技术岗必备）
- [ ] 无冗余个人信息（婚否/照片等，除非目标地区惯例要求）
- [ ] 所在地与目标岗位一致或说明意向

## B. 职业摘要（5 项）

- [ ] 有摘要/价值主张（不是"自我介绍"）
- [ ] 首屏 3 行内能让对方想继续看
- [ ] 包含量化成果或核心能力
- [ ] 与目标岗位匹配
- [ ] 不超过 3 行

## C. 工作经历（8 项）

- [ ] 强动词开头（非"负责"）
- [ ] 每条 bullet 体现"动作 + 产物 + 结果"
- [ ] 关键经历有量化数字
- [ ] STAR 法则结构完整
- [ ] 每段项目有上下文描述（系统定位+服务对象）
- [ ] 经历顺序与目标岗位匹配度高的在前
- [ ] 无时间断裂或矛盾
- [ ] 无同质化堆叠（三段写成一样）

## D. 教育背景（4 项）

- [ ] 学位/专业/时间完整
- [ ] GPA 策略（≥3.5 写，<3.5 不写）
- [ ] 相关课程（应届生/转行用）
- [ ] 无冗余（高中经历删除，除非刚毕业）

## E. 技能清单（5 项）

- [ ] 分类呈现（技术栈/工具/软技能）
- [ ] 无过时技能（如 Word 除非岗位要求）
- [ ] 与 JD 匹配
- [ ] "精通"项有项目证据支撑
- [ ] 无堆砌（控制在 10-15 项核心）

## F. 语法与表达（5 项）

- [ ] 无错别字/拼写错误
- [ ] 时态一致（过去经历用过去时）
- [ ] 避免第一人称（"我"过多）
- [ ] 中英文混排有空格
- [ ] 每句话后有句号（中文简历）

## G. 格式与排版（5 项）

- [ ] 间距/字体一致
- [ ] 篇幅合适（应届 1 页，社招 1-2 页）
- [ ] 标题层级清晰
- [ ] 对齐整齐
- [ ] 打印效果良好（`print-color-adjust: exact`）

## H. ATS 兼容性（5 项）

- [ ] 标准标题（"工作经历"而非"我的历程"）
- [ ] 关键词密度充足（JD 硬技能词原样出现）
- [ ] 无表格/图片/文本框承载关键信息
- [ ] PDF 可复制文本（非扫描件）
- [ ] 字体为通用字体（非嵌入式特殊字体）

## 整体审计（额外）

- **职业故事线**：路径是否连贯，跳槽/转岗有无合理逻辑。
- **一致性**：时间、技术栈、角色、数据规模前后无矛盾。
- **无效内容**：玩具项目、低信息量 bullet、与目标无关内容。

## 版本专属审计项

通用 8 大类之外，按目标雇主版本叠加专属审计项（soe 政治面貌/籍贯/证书/稳定；internet GitHub/项目上下文/业务量化；foreign 全英文/无隐私字段/action verbs/1 页）。完整清单见 `employer-variants.md` 的「版本专属审计项」节。

诊断时先确认版本，逐项核对版本专属项，触发即标 ⚠️ 或 ❌。
